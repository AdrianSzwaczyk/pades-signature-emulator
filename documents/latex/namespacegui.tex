{\footnotesize\ttfamily gui.\+generate\+\_\+and\+\_\+save\+\_\+keys (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}



Main logic for GUI, data validation, key generation, encryption and keys saving. 

This function retrieves user inputs from the GUI, which includes\+:
\begin{DoxyItemize}
\item PIN to encrypt the private key
\item USB path where the private key will be saved
\item Public key save location
\item Key name for the generated keys Those inputs are validated and if error occurs, it shows a specific error message.
\end{DoxyItemize}

If the data is correct, it proceeds to generate RSA keys using the \doxylink{namespacekey__generation_ae3c417929d3fd7844f3de9d675ddeafc}{generate\+\_\+rsa\+\_\+keys()} function from \doxylink{key__generation_8py}{key\+\_\+generation.\+py} file. The generated keys are then encrypted using the \doxylink{namespacekey__encryption_aae51bf239682808787b2bcc9306d187b}{encrypt\+\_\+private\+\_\+key()} function from \doxylink{key__encryption_8py}{key\+\_\+encryption.\+py} file, where the PIN is used to derive the encryption key. After encryption, the keys are saved using the \doxylink{namespacekey__saving_af528c23f394abadd10446e40f9f2af91}{save\+\_\+keys()} function from \doxylink{key__saving_8py}{key\+\_\+saving.\+py} file, where the private key is saved on the USB device and the public key is saved in the specified path.

During the process, the GUI is updated to show progress bar and status messages. If any step fails, an error message is displayed, so that the user can be informed about all the issues during the process.

If the keys are successfully generated, encrypted, and saved, a success message is displayed with the paths to the saved keys and information about any other action taken, such as clearing keys directory on the USB drive. \Hypertarget{namespacegui_a88fac34a7747b92ee235e3dd3f1ff99f}\label{namespacegui_a88fac34a7747b92ee235e3dd3f1ff99f}
